Wallet App php sdk Integrate with ease

This page contains the documentation for the Wallet App php software development kit

For installation of the php sdk simply run the composer command:


composer require walletapp/php-sdk

This SDK integrates the WalletApp public Api's an serves an easy to use Class based integration.


Make sure the API_KEY environment variable is set. Other wise pass the API key on each Class initialization, this can be done as follows: 


// Your php file
use Clients\Brands;
$brands = new Brands('YOUR_API_KEY');

// submit your request
...

The authentication class is the only class that does not need an api key to be able to handle requests.


It is used to generate an access token that can be used to make requests to the other classes.


It contains the following methods

  • login()
  • token()
  • scopes()

Login

The login method is used to login a member with their dashboard credentials

It can be used as follows:


// Your php file
use Clients\Authentication;
$auth = new Authentication();

// login the user
$auth->login(['email' => '', 'password' => '']);

Token

The token method is used fetch an api key which will be used for future requests

It can be used as follows:


// Your php file
use Clients\Authentication;
$auth = new Authentication();

// request a token
$auth->token(['email' => '', 'password' => '', 'permissions' => '']);

Scopes

The scopes method is used fetch a list of requestable permissions for an api key

It can be used as follows:


// Your php file
use Clients\Authentication;
$auth = new Authentication();

// fetch your scopes
$auth->scopes();

The Brands class Allows you to fetch the required brand data for your integration


It contains the following methods

  • get()

get

This method returns allow brands for the authenticated user

It can be used as follows:


// Your php file
use Clients\Brands;
$brands = new Brands();

// fetch your brands
$brands->get();

The Members class Allows import members, fetch import log data and the latest member changes


It contains the following methods

  • brand()
  • search()
  • logs()
  • export()
  • import()

Brand

This is a chainable method to set the brand that we are going to work with

It can be used as follows:


// Your php file
use Clients\Members;
$members = new Members();

// set the brand
$members->brand('brand_id')->search();

Search API

This method a specific member for the specified brand

It can be used as follows:


// Your php file
use Clients\Members;
$members = new Members();

// search for a member
$members->brand('brand_id')->search(['email' => '']);

Logs API

This method returns the logs for the latest members that were imported

It can be used as follows:


// Your php file
use Clients\Members;
$members = new Members();

// fetch the import logs
$members->brand('brand_id')->logs();

Export API

This exports the member for the specified brand and storecard

It can be used as follows:


// Your php file
use Clients\Members;

$members = new Members();
// export the members

$members->brand('brand_id')->export(['storecard' => '']);

Import API

This imports the member for the specified brand

It can be used as follows:


// Your php file
use Clients\Members;

$members = new Members();

// import the members
$members->brand('brand_id')->import(['members' => $members]);
